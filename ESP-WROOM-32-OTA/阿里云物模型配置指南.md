## 📋 问题排查步骤

### 第一步：确认代码中已包含数据发送功能 ✅

**代码位置：** `firmware/ESP-WROOM-32-OTA/components/BSP/MQTT/MQTT.c`

```c
// 第152-184行：publish_sensor_data 函数
void publish_sensor_data(uint16_t ir, uint16_t ps, uint16_t als)
{
    // 构建JSON数据
    cJSON *root = cJSON_CreateObject();
    cJSON *params = cJSON_CreateObject();

    cJSON_AddNumberToObject(params, "IR", ir);   // 温度
    cJSON_AddNumberToObject(params, "ps", ps);   // 湿度
    cJSON_AddNumberToObject(params, "als", als); // 光照强度

    cJSON_AddItemToObject(root, "params", params);
    cJSON_AddStringToObject(root, "method", "thing.event.property.post");

    // 发布到阿里云
    esp_mqtt_client_publish(client, TOPIC_PROP_POST, json_str, 0, 1, 0);
}
```

**结论：** 代码中**已经有完整的发送逻辑**，不是只做了连接！

## ❌ 常见问题排查

### 问题1：阿里云收不到数据

**可能原因：**
1. 物模型属性标识符不匹配
   - 代码中使用：`IR`、`ps`、`als`
   - 阿里云也必须用相同的标识符

2. Topic不正确
   - 正确的发布Topic：`/sys/k1jy7ZTTQ8V/ESP32-WROOM-32/thing/event/property/post`
   - 这个Topic是物模型的标准Topic，不需要手动创建

3. 设备未上线
   - 检查ESP32是否成功连接到阿里云
   - 查看设备运行状态是否为"在线"

### 问题2：串口显示发送成功，但阿里云没数据

**排查步骤：**
1. 检查阿里云物模型是否已发布上线
2. 检查属性标识符是否完全一致（大小写敏感）
3. 检查数据类型是否匹配
4. 查看阿里云日志服务（如果开启了）

### 问题3：数据格式错误

**正确的JSON格式：**
```json
{
  "params": {
    "IR": 255,
    "ps": 650,
    "als": 850
  },
  "method": "thing.event.property.post"
}
```

## 📊 完整的ESP32数据发送流程

```
ESP32启动
    ↓
WiFi连接
    ↓
MQTT连接阿里云
    ↓
订阅主题：/sys/.../thing/service/property/set
    ↓
每3秒发送一次数据：
    - 温度：IR (25.0~25.9℃)
    - 湿度：ps (60.0~61.9%)
    - 光照：als (800~849 lx)
    ↓
发布到Topic: /sys/.../thing/event/property/post
    ↓
阿里云接收并更新设备影子
```

   - 确认返回的msg_id是否大于0
