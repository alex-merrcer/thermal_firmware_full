### 第二步：检查输出中的关键信息

#### ✅ 如果看到"✓ [数据发布成功] 消息 ID=xxx"
说明ESP32已经成功发送数据到阿里云。

**此时去阿里云查看：**
1. 登录阿里云物联网平台
2. 进入设备详情页
3. 点击 **设备影子** → **查看影子**
4. 查看JSON中是否有数据

#### ❌ 如果看到"✗ [数据发布失败] msg_id=xxx"
说明MQTT发布失败，可能原因：
- MQTT连接已断开
- Topic不正确
- 阿里云物模型未配置

### 第三步：检查阿里云物模型配置

从你的截图看，配置基本正确，但需要确认：

#### 1. 物模型是否已发布上线？
- 进入产品 → 物模型
- 查看右上角是否有 **"已发布"** 状态
- 如果是"草稿"状态，点击 **发布上线**

#### 2. 属性标识符是否完全一致？
代码中使用：
```c
"IR"   // 温度
"ps"   // 湿度
"als"  // 光照
```

阿里云配置也必须是：
- `IR` (大小写敏感！)
- `ps` (小写！)
- `als` (小写！)

#### 3. 检查设备是否在线
- 进入设备管理 → 设备
- 查看设备状态是否为 **在线**
- 如果显示离线，说明MQTT连接有问题

### 第四步：查看阿里云日志

#### 方法1：设备影子
1. 进入设备详情页
2. 点击 **设备影子**
3. 点击 **查看影子**
4. 查看JSON数据：

```json
{
  "reported": {
    "IR": 255,
    "ps": 650,
    "als": 850
  }
}
```

如果有这个数据，说明阿里云已经收到了！

#### 方法2：物模型数据
1. 进入设备详情页
2. 点击 **物模型数据**
3. 选择 **属性数据**
4. 查看是否有记录

#### 方法3：日志服务（如果开启了）
1. 进入产品详情
2. 点击 **监控运维** → **日志服务**
3. 查看设备上报日志

## 🎯 常见问题

### 问题1：ESP32显示发送成功，但阿里云没数据

**可能原因：**
1. Topic不正确
   - 正确的Topic：`/sys/k1jy7ZTTQ8V/ESP32-WROOM-32/thing/event/property/post`
   - 检查MQTT.h中的定义

2. 物模型未发布
   - 进入物模型 → 发布上线

3. 阿里云服务器延迟
   - 等待10-30秒再查看

### 问题2：msg_id返回负数

**原因：** MQTT发布失败
**解决：**
- 检查MQTT连接状态
- 检查Topic格式
- 重启ESP32重新连接

### 问题3：设备影子是空的

**原因：** 数据格式不匹配
**检查：**
1. JSON格式是否正确
2. 属性标识符是否一致
3. 数据类型是否匹配

---

## 📊 正确的数据格式

ESP32发送的JSON应该是：
```json
{
  "params": {
    "IR": 255,
    "ps": 650,
    "als": 850
  },
  "method": "thing.event.property.post"
}
```

阿里云设备影子应该是：
```json
{
  "reported": {
    "IR": 255,
    "ps": 650,
    "als": 850
  }
}
```
